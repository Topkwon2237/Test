### Context
개방형 워크플로우 자동화 도구인 n8n을 사용하여 첫 번째 자동화 워크플로우를 구축하는 과정은 다양한 비즈니스 프로세스 및 개인 작업을 자동화하는 데 있어 중요한 첫걸음입니다. 이 문서는 n8n의 기본적인 워크플로우 생성 및 웹훅(Webhook) 기반의 응답 자동화 방법을 설명하며, 개발자가 쉽게 자동화 시스템을 구축하고 테스트할 수 있도록 돕는 데 목적이 있습니다.

### Core
다음은 n8n에서 웹훅 요청을 받아 간단한 JSON 응답을 반환하는 기본적인 워크플로우 예시입니다. 이 워크플로우는 `Webhook` 트리거 노드로 시작하여 들어오는 HTTP 요청을 감지하고, `Respond to Webhook` 노드를 통해 미리 정의된 응답을 보냅니다.

```json
{
  "nodes": [
    {
      "parameters": {},
      "name": "Webhook",
      "type": "n8n-nodes-base.webhook",
      "typeVersion": 1,
      "position": [700, 300]
    },
    {
      "parameters": {
        "responseMode": "lastNode",
        "responseBody": "{\"message\": \"Hello from n8n!\", \"status\": \"success\"}",
        "contentType": "application/json"
      },
      "name": "Respond to Webhook",
      "type": "n8n-nodes-base.respondToWebhook",
      "typeVersion": 1,
      "position": [900, 300]
    }
  ],
  "connections": {
    "Webhook": {
      "main": [
        [
          {
            "node": "Respond to Webhook",
            "type": "main",
            "index": 0
          }
        ]
      ]
    }
  }
}
```

**워크플로우 설정 가이드:**
*   `Webhook` 노드를 워크플로우의 시작점으로 추가합니다.
*   `Webhook` 노드의 URL을 복사하여 외부에서 HTTP 요청을 보낼 수 있도록 준비합니다.
*   `Respond to Webhook` 노드를 추가하고, `Response Body`에 반환하고자 하는 JSON 데이터를 입력합니다. 예시에서는 `{"message": "Hello from n8n!", "status": "success"}`를 사용합니다.
*   `Content Type`을 `application/json`으로 설정하여 JSON 형식의 응답임을 명시합니다.
*   워크플로우를 활성화(Activate)하고, `Webhook` 노드의 URL로 테스트 요청을 보냅니다.

### Insight
n8n에서 첫 번째 워크플로우를 구축하는 것은 자동화의 기본 원리를 이해하는 데 매우 중요합니다. `Webhook` 노드는 외부 시스템과의 연동을 위한 강력한 트리거 역할을 하며, `Respond to Webhook` 노드는 요청에 대한 즉각적인 피드백을 제공합니다. 이 간단한 구조는 더 복잡한 데이터 처리, 다른 서비스와의 통합, 조건부 로직 적용 등으로 확장될 수 있습니다. 워크플로우를 구축한 후에는 반드시 테스트를 통해 예상대로 동작하는지 확인해야 합니다. n8n은 시각적인 인터페이스를 제공하여 비기술 직군 사용자도 쉽게 자동화를 구현할 수 있도록 돕습니다.
