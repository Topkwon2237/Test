### Context
사용자로부터 "두번째"라는 단편적인 입력이 주어졌습니다. 기술 문서를 생성하기 위해서는 문서의 주제, 핵심 내용, 학습 배경 등 보다 구체적인 정보가 필요합니다. 본 문서는 효과적인 기술 문서 생성을 위한 정보 제공 가이드를 목적으로 합니다.

### Core
기술 문서를 효과적으로 작성하기 위해 다음 정보를 포함하여 입력해 주십시오:

*   **문서의 주요 주제 또는 제목:** 어떤 기술에 대한 문서인가요?
*   **학습 배경 또는 해결하려는 문제:** 왜 이 문서를 작성하게 되었나요? 어떤 문제를 해결하려 했나요?
*   **핵심 내용 (코드 스니펫, 로직, 설정값 등):** 실제 적용 가능한 코드, 설정, 개념 설명 등을 Markdown 코드 블록을 활용하여 구체적으로 포함해 주세요.
*   **인사이트 또는 결과:** 이 기술을 통해 얻은 결과나 개인적인 견해를 명확하게 작성해 주세요.

**예시 입력:**
```
- 메모: 파이썬 딕셔너리 값으로 정렬
- 문제: 파이썬에서 딕셔너리를 값 기준으로 정렬하는 효율적인 방법이 필요함.
- 코드:
  ```python
  my_dict = {'apple': 3, 'banana': 1, 'cherry': 2}
  sorted_items = sorted(my_dict.items(), key=lambda item: item[1])
  sorted_dict = {k: v for k, v in sorted_items}
  print(sorted_dict)
  ```
- 결과: `sorted()` 함수와 람다 표현식을 활용하여 딕셔너리를 값 기준으로 쉽게 정렬할 수 있었으며, 이는 데이터 처리 시 유용하게 활용될 수 있음을 확인했습니다.
```

### Insight
AI 기반 문서 작성 에이전트는 사용자의 의도를 정확히 파악하고 고품질의 기술 문서를 생성하기 위해 명확하고 구체적인 지시(프롬프트)를 필요로 합니다. "두번째"와 같은 모호하고 단편적인 입력은 문서 내용의 추론이나 생성을 어렵게 합니다. 따라서 문서의 핵심 요소를 포함하는 구조화된 정보를 제공함으로써, 에이전트가 사용자의 의도에 부합하는 정확하고 완결성 높은 기술 문서를 작성할 수 있도록 유도해야 합니다. 이는 프롬프트 엔지니어링(Prompt Engineering)의 핵심 원칙 중 하나입니다.
